# Accessing and Managing Projects from the Dashboard

Your [dashboard](../../getting-started/working_with_the_dashboard) is a versatile hub that can be customized to contain important interactive blocks for your projects, forms and records. The project block, in particular, makes it easy to access the main page of projects that you work with regularly. It also makes the performing of certain Kora actions for projects, just one click away. What follows is a description of how to use project blocks from your dashboard to manage a project.

## The Project Block

A project block within your dashboard will contain the name of a project and its description. Click on your project name to access that project's main *Project* page.

<img style="display:block;margin:auto;max-width:100%" src="../projects-img/managing_projects_from_the_dashboard_1_annotated.png" title="Step 1">

Below the project name and description are the **Quick Action** icons. The functions of these **Quick Action** icons are summarized below then linked to relevant documentation. For instructions on how to reorder your *Quick Actions**, see the documentation on [working with the dashboard](../../getting-started/working_with_the_dashboard).

<img style="display:block;margin:auto;max-width:100%" src="../projects-img/managing_projects_from_the_dashboard_2_annotated.png" title="Step 2">

## Project Quick Actions

This section lists the types of Quick Actions available to the project block, gives a short description, and provides a link to any further documentation. There are 9 **Quick Action** options available for a project block:

### Edit Project

This brings you to the *Edit Project* page where you can change the title and description of your project. Check out the documentation on [editing a project](../editing_a_project)

<img style="display:block;margin:auto;max-width:100%" src="../projects-img/managing_projects_from_the_dashboard_3_annotated.png" title="Step 3">

### Search Project Records

This allows you to search through the records within a project. Check out the documentation on [searching and browsing records](../../records/searching_and_browsing_records)

<img style="display:block;margin:auto;max-width:100%" src="../projects-img/managing_projects_from_the_dashboard_4_annotated.png" title="Step 4">

### Create New Form

This allows you to create a new form for the project in the project block. See the documentation on [creating a form](../../forms/creating_a_form)

<img style="display:block;margin:auto;max-width:100%" src="../projects-img/managing_projects_from_the_dashboard_5_annotated.png" title="Step 5">

### Import Form

This brings you to the page for importing forms into a project. Check out the documentation on [importing a form](../../forms/importing_a_form).

<img style="display:block;margin:auto;max-width:100%" src="../projects-img/managing_projects_from_the_dashboard_6_annotated.png" title="Step 6">

### Project Permissions

This allows administrators to give Kora users the ability to create, edit and delete forms. See section on [editing project permissions](../editing_a_project) in the editing projects documentation. <p align="center"> <img src="../projects-img/managing_projects_from_the_dashboard_7_annotated.png" width="100%" style="align:center" title="Step 7"> </p>

### Field Value presets

These allow you to create predefined field options using lists or regex. For more information, see the documentation on [field value presets](../../forms/field_value_presets).

<img style="display:block;margin:auto;max-width:100%" src="../projects-img/managing_projects_from_the_dashboard_8_annotated.png" title="Step 8">

### Import Multi-Form Records Setup

This allows you to import records into multiple forms within a project. See the documentation on [importing multi form records](../../records/importing_multi_form_records).

<img style="display:block;margin:auto;max-width:100%" src="../projects-img/managing_projects_from_the_dashboard_9_annotated.png" title="Step 9">

### Kora 2 Scheme Importer

This allows you to import records from a previous version of the Kora installation. Contact us or create an issue on the [official Kora GitHub repository](https://github.com/matrix-msu/kora/issues/new/choose) for help if you need to import records in this way.

<img style="display:block;margin:auto;max-width:100%" src="../projects-img/managing_projects_from_the_dashboard_10_annotated.png" title="Step 10">

### Export Project

This allows you to save the project as a .kProj file onto your computer. Check out the documentation on [exporting projects](../exporting_a_project).

<img style="display:block;margin:auto;max-width:100%" src="../projects-img/managing_projects_from_the_dashboard_11_annotated.png" title="Step 11">
